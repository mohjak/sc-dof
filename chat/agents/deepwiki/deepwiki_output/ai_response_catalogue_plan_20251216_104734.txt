1. Repository Type Detection
The repository is a Frontend Application. It is a React-based web application with TypeScript, using Vite as the build tool, and Tailwind CSS for styling. The application is designed as a multilingual event platform for the Syrian community, featuring responsive design and internationalization support.

2. Codebase Overview/Analysis
- Core functionality: A multilingual event website for the Syrian community with event details, agenda, speakers, registration, and organizational information.
- Primary use cases: Displaying event information in Arabic, English, and Turkish with RTL support for Arabic.
- Primary programming languages: TypeScript, JSX, CSS (via Tailwind)
- Critical frameworks and library dependencies:
  - React 18 with TypeScript
  - Vite as build tool
  - Tailwind CSS for styling
  - ShadCN UI components built on Radix UI primitives
  - React Router DOM for routing
  - React Query for data management
  - date-fns for date manipulation

3. Project Directory Architecture Mapping/Analysis
- src/: Main source code directory
  - components/: Reusable UI components including ShadCN UI library components and custom application components
  - data/: Event data and content
  - hooks/: Custom React hooks
  - lib/: Utility functions
  - pages/: Application pages
- public/: Static assets and manifest files
- Configuration files:
  - package.json: Project metadata and dependencies
  - vite.config.ts: Vite build configuration
  - tailwind.config.ts: Tailwind CSS customization
  - tsconfig.json: TypeScript configuration
  - eslint.config.js: ESLint configuration

4. Core Functionality and Services Identification
Principal features:
- Multilingual support (Arabic, English, Turkish) with RTL layout for Arabic
- Event information display (date, time, location)
- Agenda section with event schedule
- Speakers section with participant information
- Goals and purpose of the event
- Organizers information
- Registration section
- Responsive design with mobile support
- Internationalization system with language context

5. Codebase Deep Dive
Critical code modules:
- LanguageContext.tsx: Manages multilingual functionality with translation system and RTL support
- App.tsx: Root application component with providers for routing, internationalization, and state management
- Index.tsx: Main page component that composes all sections
- eventData.ts: Central data model for event information
- ui/ components: ShadCN UI library components built on Radix UI primitives

Architectural patterns:
- Component-based architecture with React
- Context API for global state management (language, theme)
- Atomic design principles in UI components
- Custom hooks for reusable logic (use-mobile, use-toast)
- Provider pattern for dependency injection

8. Code Structure Analysis
Code structure patterns:
- Component composition pattern
- Context-based state management
- Custom hooks for reusable logic
- Separation of concerns between UI components and business logic

Component relationships:
- App component wraps all providers (QueryClient, LanguageProvider, TooltipProvider)
- Index page imports and composes all section components
- Section components use UI components from the ui/ directory
- LanguageContext provides translation function (t) and language state to all components

9. Data Flow Analysis
Data pathways:
- LanguageContext provides language state and translation function to all components
- eventData.ts provides event data to components
- Translation system flows from LanguageContext to individual components via the t() function
- User interactions (language selection) update context state which propagates to all components

Critical data models:
- LanguageContextType: Defines the structure of the language context
- eventData: Central data object containing all event information
- Speaker interface: Defines speaker data structure

Extension mechanisms:

Integration points:

11. Dependency Analysis
External dependencies:
- React and React DOM: Core UI library
- Vite: Build tool and development server
- Tailwind CSS: Styling framework
- ShadCN UI: Component library built on Radix UI
- React Router DOM: Client-side routing
- React Query: Data fetching and caching
- date-fns: Date manipulation utilities
- lucide-react: Icon library

Internal component interdependencies:
- App depends on all section components and providers
- Section components depend on UI components and hooks
- UI components depend on utility functions (cn)
- All components depend on LanguageContext for translations

12. User Workflow Mapping
Common user scenarios:
1. User visits site → Language selection → Views event information in selected language
2. User navigates through event sections using navbar or scrolling
3. User views event details (date, time, location) and clicks to view on map
4. User explores agenda, speakers, and organizers
5. User registers for event through registration section
6. User switches languages at any point to view content in preferred language

Key entry points:
- Main page (Index) as primary entry point
- Direct links to specific sections (agenda, speakers, etc.)
- Language switcher for immediate language selection
